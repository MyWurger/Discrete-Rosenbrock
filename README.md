# Discrete-Rosenbrock

Графическое приложение на C++/Qt для исследования дискретного метода Розенброка в задачах безусловной оптимизации. Проект реализует сам алгоритм, разбор пользовательских функций через `muparser`, подробную трассировку итераций и визуализацию траектории поиска для двумерных и трехмерных задач.

Репозиторий подготовлен как лабораторная работа по теме численных методов оптимизации и ориентирован не только на получение ответа `x*`, но и на анализ поведения метода: изменения шагов `Delta_j`, удачных и неудачных пробных переходов, перестроения базиса направлений и сравнения найденного численного решения со стационарной точкой квадратичной функции.

## Что умеет проект

- Решает задачи безусловной минимизации дискретным методом Розенброка.
- Принимает функцию в текстовом виде и строит вычислимый объект через `muparser`.
- Поддерживает размерность до 3 переменных: `x1`, `x2`, `x3`.
- Автоматически определяет размерность по введенному выражению.
- Показывает полную таблицу трассировки по шагам метода.
- Строит отдельное окно визуализации:
  - 2D для функций двух переменных;
  - 3D для функций трех переменных.
- Синхронизирует график с таблицей итераций.
- Экспортирует таблицу трассировки в PNG.
- Для квадратичных функций дополнительно оценивает стационарную точку через условие `grad f = 0` и сравнивает ее с численным результатом.

## Реализованный метод

В проекте реализован дискретный метод Розенброка с такими ключевыми этапами:

1. Старт из начальной точки `x0`.
2. Последовательный проход по текущему базису направлений.
3. Пробные шаги вида `y_j + Delta_j * s_j`.
4. Растяжение шага при успехе и сжатие при неудаче.
5. Перестройка базиса направлений после завершения внешней итерации.
6. Остановка при выполнении критерия `|Delta_j| < epsilon` для всех направлений либо по защитному лимиту числа итераций.

Основные параметры метода по умолчанию:

- `epsilon = 1e-3`
- `alpha = 1.5`
- `beta = -0.5`
- `Delta_0 = 0.1`
- `maxIterations = 50000`

## Тестовые функции

В коде присутствуют две стандартные функции из лабораторного задания:

### F1

```text
9*x1^2 + 16*x2^2 - 90*x1 - 128*x2
```

Стартовая точка по умолчанию:

```text
7 -2
```

### F2

```text
x1^2 + 2*x1*x2 + 2*x2^2 + x3^2 - x2*x3 + x1 + 3*x2 - x3
```

Стартовая точка по умолчанию:

```text
0 0 0
```

При запуске GUI по умолчанию подставляется именно `F2`.

## Технологии и зависимости

- `C++17`
- `CMake >= 3.15`
- `Qt6 Widgets`
- `Qt6 Charts` для 2D-визуализации
- `Qt6 DataVisualization` для 3D-визуализации
- встроенная в репозиторий библиотека `muparser`

Замечания по зависимостям:

- `Qt6 Widgets` обязателен.
- `Qt6 Charts` и `Qt6 DataVisualization` подключаются условно.
- Если соответствующий Qt-модуль не найден, проект всё равно собирается, но часть визуализации будет недоступна.

## Сборка

### macOS + Homebrew

Если Qt установлен через Homebrew, CMake в этом проекте умеет самостоятельно искать:

- `qtbase`
- `qtcharts`
- `qtdatavis3d`

Типовая последовательность:

```bash
brew install cmake qtbase qtcharts qtdatavis3d
cmake -S . -B build
cmake --build build
```

### Универсальный вариант

Если Qt установлен вручную или находится в нестандартном месте, можно явно передать пути в CMake:

```bash
cmake -S . -B build \
  -DQt6_DIR=/path/to/Qt6/lib/cmake/Qt6 \
  -DQt6Charts_DIR=/path/to/Qt6Charts/lib/cmake/Qt6Charts \
  -DQt6DataVisualization_DIR=/path/to/Qt6DataVisualization/lib/cmake/Qt6DataVisualization

cmake --build build
```

## Запуск

После сборки исполняемый файл создается с именем `Lab_2`.

Пример запуска:

```bash
./build/Lab_2
```

На macOS в проекте предусмотрена дополнительная логика поиска Qt platform plugins, чтобы приложение корректно находило `cocoa` plugin при запуске.

## Как пользоваться приложением

1. Введите математическое выражение функции.
2. Укажите стартовую точку в формате списка координат через пробел.
3. Задайте параметры `epsilon`, `alpha`, `beta` и начальный шаг.
4. Нажмите кнопку расчета.
5. Изучите итоговую точку `x*`, значение `f(x*)`, число итераций и число вычислений функции.
6. Просмотрите таблицу трассировки.
7. При необходимости откройте окно графика и/или экспортируйте таблицу в PNG.

## Формат ввода функции

Парсер поддерживает:

- переменные `x1`, `x2`, `x3`
- краткую запись `x` как синоним `x1`
- константы `pi` и `e`
- стандартные операции и функции, доступные в `muparser`

Ограничения:

- размерность должна быть от 1 до 3;
- все вычисления должны давать конечные значения;
- если выражение содержит синтаксическую ошибку, GUI показывает понятное сообщение с позицией и проблемным токеном.

## Формат ввода стартовой точки

Координаты задаются через пробел, например:

```text
1 2
```

или

```text
0 0 0
```

Количество координат должно совпадать с размерностью функции, определенной по выражению.


Ключевые модули:

- `RosenbrockDiscreteMethod.*` — ядро численного метода.

- `MainWindow.*` — главное окно, валидация ввода, таблица результатов, экспорт.
- `Matplot2DViewer.*` и `Matplot3DViewer.*` — визуализация траектории метода.

## Что попадает в git

В репозиторий включены исходники, конфигурация CMake, отчетные и тестовые материалы. Каталог сборки и временные артефакты исключены через `.gitignore`.

## Возможные причины проблем при сборке

- Не найден `Qt6 Widgets`.
- Не установлены `Qt6 Charts` или `Qt6 DataVisualization`, из-за чего недоступна часть визуализации.
- CMake не знает путь к Qt, если он установлен вне стандартных каталогов.
- На macOS не найдены Qt platform plugins.

В большинстве случаев это решается явной передачей путей `Qt6_DIR`, `Qt6Charts_DIR` и `Qt6DataVisualization_DIR` при конфигурации CMake.
